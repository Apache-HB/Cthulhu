    [terms]

    1. undefined behavior
      [terms.undefined]

      behavior for which this document imposes no requirements.

      unspecified behavior
      [terms.unspecified]

      behavior, for a well-formed program construct and correct data, that depends on implementation.

      abstract machine
      [terms.abstract]

      the semantic descriptions in this document define a parameterized nondeterministic abstract machine. this document places no requirement on the structure of conforming implementations. in particular, then need not copy or emulate the structure of the abstract machine. rather, conforming implementations are required to emulate only the observable behavior of the abstract machine as described in [intro.terms.observable]
